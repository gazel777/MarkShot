指定されたファイルまたはディレクトリのコードレビューを行う。

対象: $ARGUMENTS

## レビュー観点

### 1. コード品質
- 可読性（命名、構造、コメント）
- 重複コードの有無
- 適切な抽象化

### 2. セキュリティ
- インジェクション脆弱性（SQL、XSS、コマンド）
- 認証・認可の実装
- 機密情報のハードコード

### 3. パフォーマンス
- N+1問題
- 不要なループ・再計算
- メモリリーク

### 4. エラーハンドリング
- 例外処理の適切さ
- エラーメッセージの明確さ
- 境界値の考慮

## 出力形式

各問題について以下の形式で報告:

```
### [重要度: 高/中/低] 問題のタイトル
- ファイル: path/to/file.ts:行番号
- 問題: 何が問題か
- 提案: どう修正すべきか
```

## 注意事項

- 重要度「高」のセキュリティ問題は最優先で報告
- 改善提案は具体的なコード例を含める
- 良い実装があれば肯定的なフィードバックも含める
