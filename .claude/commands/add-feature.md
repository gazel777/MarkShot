新機能を追加する。

機能の説明: $ARGUMENTS

## 手順

### 1. 計画フェーズ（Planモード推奨）
- 既存コードベースの構造を把握
- 影響範囲を特定
- 実装方針を策定

### 2. ブランチ作成
```bash
git switch -c feature/機能名
```

### 3. 実装フェーズ

#### テスト駆動で進める場合（推奨）
1. テストファイルを作成
2. 失敗するテストを書く
3. テストが通る最小限の実装
4. リファクタリング

#### 通常の実装
1. 必要なファイル・コンポーネントを作成
2. ロジックを実装
3. テストを追加

### 4. 検証フェーズ
- Linter/Formatter実行
- 全テスト実行
- 手動での動作確認

### 5. コミット・PR
- 変更をコミット
- PRを作成

## 注意事項

- 既存のコーディング規約・ディレクトリ構造に従う
- 1つのPRで1つの機能に絞る
- 大きな機能は小さなPRに分割することを検討
- セキュリティ要件を明確にしてから実装
